<task type="auto">
  <name>Task 1: Author ForgeLogo component (horizontal + icon variants)</name>
  <files>src/components/ForgeLogo.tsx</files>
  <action>
Create `src/components/ForgeLogo.tsx` — a single React component with a `variant` prop.

**Interface:**
```tsx
interface ForgeLogoProps {
  variant?: 'horizontal' | 'icon';
  className?: string;
}
export function ForgeLogo({ variant = 'horizontal', className = 'h-7 w-auto' }: ForgeLogoProps)
```

**Icon mark design (Claude's discretion per CONTEXT.md):**
Design a simple geometric mark that pairs with the word "Forge". Keep it to 1-3 paths, no gradients, scalable from 16px to 200px. Good options: a stylized anvil silhouette, an angular hammer head, a geometric "F" with forge/heat feel, or an abstract flame-and-steel shape. The mark must feel like it belongs with the word "Forge" — not a standalone generic icon.

**SVG rules from RESEARCH.md:**
- Icon-only variant: `viewBox="0 0 32 32"`, icon mark in emerald `fill="#10b981"` on transparent background
- Horizontal variant: `viewBox="0 0 140 32"`, icon mark on left, wordmark "Forge" on right in `fill="#e2e8f0"`
- NEVER set fixed `width`/`height` attributes on the `<svg>` root — use `viewBox` only, let Tailwind className control sizing
- Add `aria-label="Forge"` and `role="img"` on both SVG variants
- Use hard-coded hex values (`fill="#10b981"`, `fill="#e2e8f0"`) directly on `<path>` attributes — the logo is a brand constant, not theme-switchable

**Wordmark in horizontal variant:**
Use SVG `<text>` element or `<path>` data for "Forge". If using `<text>`: `font-family="system-ui, sans-serif" font-weight="700" font-size="18" fill="#e2e8f0"`. Position it to the right of the icon mark with appropriate spacing. Alternatively, if hand-tracing the wordmark as paths looks better, do that — it's Claude's discretion.

**Anti-patterns to avoid (from RESEARCH.md):**
- Do NOT use `fill="currentColor"` for the wordmark text — color is fixed, not inherited
- Do NOT use `fill="currentColor"` for the icon mark in auth page usage (colors are brand constants)
- Do NOT add `width`/`height` attributes to the SVG root elements

<task type="auto">
  <name>Task 2: Create icon.svg for browser favicon</name>
  <files>src/app/icon.svg</files>
  <action>
Create `src/app/icon.svg` — a plain SVG file (not JSX) containing the icon-only mark. This is the same visual shape as `ForgeLogo variant="icon"` but as a raw SVG file (no React wrapper).

**File requirements:**
- Plain SVG: `<svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 32 32">`
- Same icon mark paths as used in `ForgeLogo variant="icon"`
- Use `fill="#10b981"` (emerald) for the mark
- No React syntax, no `className`, no JSX — pure SVG XML
- File goes in `src/app/icon.svg` (NOT `public/`) — Next.js file convention for favicons requires placement in the `app/` directory

**Next.js behavior:** Once `src/app/icon.svg` exists, Next.js auto-generates `<link rel="icon" href="/icon.svg" sizes="any" type="image/svg+xml" />` in the page `<head>`. No changes to `layout.tsx` needed.

**Do NOT remove** `src/app/favicon.ico` — keep the existing ICO file as a legacy browser fallback. The new `icon.svg` coexists with it.
