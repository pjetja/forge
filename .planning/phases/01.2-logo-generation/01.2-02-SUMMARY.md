## Decisions Made

- **h-7 not h-7 w-auto:** The `important_note` in execution context confirmed default className is `h-7`; `w-auto` is redundant since the SVG carries no explicit width attribute.
- **React fragment on auth pages:** Auth layout wraps content in `min-h-screen flex items-center justify-center max-w-md`; using a div wrapper would double-center. Fragment lets logo+card sit as siblings in the layout's column flow.
- **aria-label on nav Link:** ForgeLogo's horizontal variant has `role="img" aria-label="Forge"`. The wrapping Link has no visible text, so `aria-label="Forge home"` on the Link gives screen readers a navigable label describing both the destination and brand.
- **Logo in TraineeSignupContent:** The async inner component renders the actual page content; the outer `TraineeSignupPage` just delegates to it. The fragment and logo belong in the inner component's return.

None — plan executed exactly as written, with the sizing correction from the `important_note` (h-7 not h-7 w-auto on nav logos).
