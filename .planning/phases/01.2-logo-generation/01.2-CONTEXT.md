# Phase 01.2: Logo Generation - Context

**Gathered:** 2026-03-10
**Status:** Ready for planning

<domain>
## Phase Boundary

Create a proper SVG logo for the Forge app and integrate it across the key touchpoints. The current `⚡ Forge` plain text + emoji in the nav is a placeholder — this phase replaces it with a designed logo asset. Marketing assets, animated logos, and dark/light mode variants are out of scope.

</domain>

<decisions>
## Implementation Decisions

### Logo composition
- Icon + wordmark lockup: a graphic icon that pairs visually with the "Forge" name (they belong together — the graphic is designed around the word, not independent from it)
- Two variants required: horizontal lockup (icon left, wordmark right) and icon-only
- The graphic should be a custom SVG shape — NOT the Unicode ⚡ emoji — designed to feel like it belongs with the word "Forge"
- Style guide: dark navy (#0f172a) background, emerald (#10b981) primary color — consistent with the existing design system

### Placement in this phase
- **Nav bar**: Replace the `⚡ Forge` text span in both trainer and trainee layouts with the SVG logo component
- **Auth pages**: Add the horizontal logo above the dark surface card on login and signup pages
- **Favicon**: Replace the default Next.js favicon with the icon-only variant (SVG favicon + PNG fallback)
- **Browser tab**: Update `<title>` and metadata to "Forge" (no tagline needed in this phase)

### Claude's Discretion
- Exact icon shape/design (Claude creates the SVG — user will review visually as part of checkpoint)
- Icon proportions, stroke weight, and exact emerald shade used within the mark
- Whether to use a `<ForgeIcon />` React component or inline SVG in layouts
- Favicon generation approach (SVG favicon or script to generate PNG sizes)

</decisions>

<specifics>
## Specific Ideas

- The graphic element should feel paired with "Forge" — it's not a standalone symbol, it's a logo that reads as a unit
- Existing color tokens must be used (emerald accent, dark navy) — no new colors introduced
- The logo should replace the text nav element cleanly, keeping the same visual weight in the nav bar

</specifics>
