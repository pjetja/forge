- Admin client used for profile insert in callback — reason: at the moment verifyOtp() completes, session cookies may not be readable yet by the regular server client
- emailRedirectTo uses NEXT_PUBLIC_SITE_URL env var with fallback to http://localhost:3000
- signUp() passes data: { name } in user_metadata so callback can read name for profile creation
- claimInviteToken type fixed from `any` to `Awaited<ReturnType<typeof createAdminClient>>`
