  <action>
Create the workout session page as a Server Component. Params: `{ assignedPlanId: string; sessionId: string }`.

**Data fetching:**
1. Get Supabase client + getClaims (redirect to /login if missing)
2. Fetch the workout session:
   ```
   workout_sessions WHERE id = params.sessionId AND trainee_auth_uid = claims.sub .single()
   ```
   If null → notFound()
3. Fetch the schema with exercises (join to exercises table for names):
   ```
   assigned_schemas?id=eq.{session.assignedSchemaId}
   .select('id, name, assigned_plan_id, assigned_schema_exercises(id, exercise_id, sort_order, sets, reps, target_weight_kg, per_set_weights, exercises(name, muscle_group))')
   .single()
   ```
   Sort exercises by sort_order ascending.
   Handle PostgREST array join: use Array.isArray guard for `assigned_schema_exercises` and nested `exercises`.

4. Fetch session_sets already logged for this session:
   ```
   session_sets WHERE session_id = params.sessionId SELECT assigned_schema_exercise_id, set_number, completed_at
   ```

5. For each exercise, compute: setsLogged = count of session_sets rows where assignedSchemaExerciseId matches this exercise, setsRequired = exercise.sets

**Layout:**

Back button → `/trainee/plans/${params.assignedPlanId}`

Header: Schema name (h1), plan name sub-label (fetch assigned_plan name or pass via breadcrumb)

Metadata row: "Workout {schema.slotIndex + 1}" · session date (session.startedAt formatted as "Mar 13")

Exercise list (each row is tappable — Link to exercise detail page):
- Each row: exercise name, muscle group tag, "{setsLogged}/{setsRequired} sets logged"
- If setsLogged >= setsRequired → show completed checkmark (green accent)
- If setsLogged > 0 but < setsRequired → show partial progress (e.g., yellow/in-progress indicator)
- If setsLogged === 0 → neutral state (no indicator)
- Entire row is a Link to `/trainee/plans/${assignedPlanId}/workouts/${sessionId}/exercises/${exercise.id}`

Footer (sticky or at bottom of list):
- "Finish Workout" button — only show if session.status === 'in_progress'
- If session.status === 'completed' → show "Workout completed" banner instead (trainee may browse history)

Pass to FinishWorkoutButton: sessionId, totalPlanSets (sum of all exercise.sets), loggedSetCount (sum of setsLogged across all exercises)
