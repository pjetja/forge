Create the SQL migration file exactly as specified in 04-RESEARCH.md (Database Schema section). Key requirements:

**workout_sessions table:**
- id UUID PK DEFAULT gen_random_uuid()
- assigned_schema_id UUID NOT NULL REFERENCES assigned_schemas(id) ON DELETE RESTRICT
- trainee_auth_uid UUID NOT NULL
- status TEXT NOT NULL DEFAULT 'in_progress' CHECK (status IN ('in_progress', 'completed', 'abandoned'))
  - Include 'abandoned' in the CHECK constraint (needed for abandon session flow — research open question resolved here)
- started_at TIMESTAMPTZ NOT NULL DEFAULT now()
- completed_at TIMESTAMPTZ
- notes TEXT
- created_at TIMESTAMPTZ DEFAULT now()

**session_sets table:**
- id UUID PK DEFAULT gen_random_uuid()
- session_id UUID NOT NULL REFERENCES workout_sessions(id) ON DELETE CASCADE
- assigned_schema_exercise_id UUID NOT NULL REFERENCES assigned_schema_exercises(id) ON DELETE RESTRICT
- set_number INTEGER NOT NULL (1-indexed)
- actual_reps INTEGER NOT NULL
- actual_weight_kg NUMERIC(6,2)
- muscle_failure BOOLEAN NOT NULL DEFAULT false
- notes TEXT
- completed_at TIMESTAMPTZ NOT NULL DEFAULT now()
- UNIQUE (session_id, assigned_schema_exercise_id, set_number) — enables upsert idempotency

**RLS policies (follow Phase 2/3 pattern — always pair USING with WITH CHECK for FOR ALL):**
- workout_sessions: Trainee manages own sessions (FOR ALL, USING + WITH CHECK trainee_auth_uid = auth.uid())
- workout_sessions: Trainer reads trainee sessions (FOR SELECT, using trainer_trainee_connections join)
- session_sets: Trainee manages own sets (FOR ALL, USING + WITH CHECK via subquery to workout_sessions)
- session_sets: Trainer reads trainee sets (FOR SELECT, via workout_sessions join to trainer_trainee_connections)

**Indexes:**
- idx_workout_sessions_trainee_status ON workout_sessions (trainee_auth_uid, status)
- idx_workout_sessions_schema_id ON workout_sessions (assigned_schema_id)
- idx_session_sets_session_id ON session_sets (session_id)
- idx_session_sets_exercise_completed ON session_sets (assigned_schema_exercise_id, completed_at)
