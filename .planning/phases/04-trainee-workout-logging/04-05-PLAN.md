   Use Array.isArray guard for exercises join (RESEARCH.md Pitfall 4).

   ```
   session_sets WHERE session_id = params.sessionId AND assigned_schema_exercise_id = params.exerciseId
   ORDER BY set_number
   ```

   ```
   session_sets WHERE assigned_schema_exercise_id = params.exerciseId
     AND completed_at >= prevStart AND <= prevEnd
   ```

**State management:**
Use `useOptimistic` for set completion (mark completed instantly before server confirms):
```typescript
const [optimisticSets, addOptimistic] = useOptimistic(
  sets,
  (state, update: { setNumber: number; reps: number; weight: number | null; failure: boolean }) =>
    state.map(s => s.setNumber === update.setNumber
      ? { ...s, completed: true, actualReps: update.reps, actualWeightKg: update.weight, muscleFailure: update.failure }
      : s
    )
);
```

Use local `useState` for editable reps/weight per row (controlled inputs). Initialize from set.actualReps / set.actualWeightKg.

**Per-row editable state:** Use a map: `const [editState, setEditState] = useState<Record<number, { reps: number; weight: string; failure: boolean }>>(() => Object.fromEntries(sets.map(s => [s.setNumber, { reps: s.actualReps, weight: s.actualWeightKg?.toString() ?? '', failure: s.muscleFailure }])))`

**Set row layout (each row in the list):**
```
| # | Reps [input] | Weight [input] kg | ☐ Failure | ✓ Complete | Last week |
```

- Set number: small label (e.g., "Set 1")
- Reps input: type="number" min="1" max="100", shows editState[setNumber].reps — full-width on mobile friendly (inputmode="numeric")
- Weight input: type="number" step="0.5", shows editState[setNumber].weight — optional (empty = bodyweight)
- Muscle failure checkbox: toggles editState[setNumber].failure
- Complete button (checkmark icon): calls handleComplete for this set
  - When set is already completed (optimistic or server-confirmed): show green checkmark, row is visually greyed (opacity-60)
  - Do not block interaction — trainee can re-tap to update values and re-save (upsert handles it)
- Last week column: compact display "8×80kg" or "—" (if no lastWeek data)
  - If lastWeekFailure: append "🔴" or a small red failure indicator
  - Styling: text-xs text-text-secondary (muted, secondary information)

**handleComplete function:**
```typescript
const [isPending, startTransition] = useTransition();

const handleComplete = (setNumber: number) => {
  const edit = editState[setNumber];
  const weightVal = edit.weight === '' ? null : parseFloat(edit.weight);
  startTransition(async () => {
    addOptimistic({ setNumber, reps: edit.reps, weight: weightVal, failure: edit.failure });
    const result = await completeSet({
      sessionId,
      assignedSchemaExerciseId: exerciseId,
      setNumber,
      actualReps: edit.reps,
      actualWeightKg: weightVal,
      muscleFailure: edit.failure,
    });
    if ('error' in result) {
      // Show inline error — toast not available, use a simple state
      console.error(result.error);
      // The optimistic update will be reverted by React on error throw,
      // but since we don't throw here, add a local error state
    }
  });
};
```

**+Add set button:**
