## What Was Built

### docs/design-tokens.md

Structured token reference extracted from `src/app/globals.css` `@theme inline`. Contains:
- All 8 color tokens with exact hex values, CSS variable names, inline HTML color swatches, and usage descriptions
- Typography section: Lato font details, weights (400/700), full Tailwind scale (text-xs through text-2xl) with sample text
- ForgeLogo brand colors section documenting hard-coded values in the ForgeLogo component
- Figma sync instructions for manual token updates

### docs/FIGMA-WORKFLOW.md

Design workflow guide for the full Figma library. Contains:
- `FIGMA_LINK_PLACEHOLDER` for the user to fill in after creating the Figma file
- Library structure documentation (5 pages: Tokens, Atoms, Molecules, Organisms, Phase 2: Exercise Library) with component inventory per page
- Step-by-step sub-phase usage guide (how to use the library for each new UI phase)
- Component naming conventions (slash grouping, variant property format)
- Token sync instructions
- Bootstrap script usage instructions
- MCP setup outcome documented (read-only, Plugin API is primary path)

### docs/figma-bootstrap.js

Runnable Figma Plugin API script. Features:
- Top-level async IIFE (`(async () => { ... })()`) for await support
- Font preloading (`figma.loadFontAsync`) before any text operations
- Renames Page 1 to "Tokens" (never removes it — Figma requires at least 1 page)
- Creates 4 additional pages using `figma.createPage()`
- All page switches use `await figma.setCurrentPageAsync()` (no deprecated sync setter)
- 6 `combineAsVariants()` calls for: Buttons/Primary (4 variants), Buttons/Secondary (3), Buttons/Ghost (2), Inputs/Text (4), Inputs/Textarea (4), Badges/Tag (2)
- 10 single components: Logo/Horizontal, Logo/Icon, Forms/InputGroup, Forms/ErrorMessage, Cards/Shell, Dialogs/ModalShell, Search/SearchBar, Navigation/TrainerNav, Navigation/TraineeNav, DataDisplay/Table, DataDisplay/EmptyState, Exercise/Card, Exercise/SearchBar, Exercise/CreateForm, Exercise/DetailView
- `layoutMode` set before `primaryAxisSizingMode`/`counterAxisSizingMode`/`itemSpacing`/padding (required ordering)
- Syntax verified clean with `node --check`
- Closes with component count message

## Authentication Gates / MCP Outcome

**Task 1 (checkpoint:human-action):** Figma MCP investigated by user.

**Outcome:** Official Figma MCP (`mcp.figma.com`) is read-only for component authoring. Available write tools (`generate_figma_design`, `generate_diagram`, `send_code_connect_mappings`, `create_design_system_rules`) cannot create arbitrary component nodes.

**Impact on plan:** Documented in `docs/FIGMA-WORKFLOW.md` MCP section. Plugin API bootstrap script (`docs/figma-bootstrap.js`) is the primary path for programmatic library creation. `generate_figma_design` noted as potentially useful for importing live app screens in future sub-phases.
