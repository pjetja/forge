## Summary

This phase creates a Figma component library (the "Forge" design system) for use as a solo design reference before coding future phases. The work spans two tracks: (1) Claude-executed code tasks — generating documentation files and a Figma plugin bootstrap script — and (2) user-executed Figma tasks with human-verify checkpoints for visually building components.

The critical first-task is investigating Figma MCP automation. Research confirms that the **official Figma remote MCP server** works on the free Starter plan and can be set up with a single Claude Code command. However, the official MCP is primarily a read + design-to-code bridge, not a general-purpose component creator. The **`claude-talk-to-figma-mcp`** community server (v0.9.2, updated February 2026) provides write access including creating frames, rectangles, text, and manipulating styles — but it requires the Figma desktop app, a local WebSocket server, and a plugin installed in Figma. It does not expose a dedicated `create_component` or `create_page` command in its current toolset.

The **Figma Plugin API bootstrap script** (plain JavaScript run inside Figma's built-in console or as a one-time plugin) is the most reliable fallback. It has full access to `figma.createPage()`, `figma.createComponent()`, `figma.combineAsVariants()`, and auto-layout APIs, and runs on the free plan with no external dependencies. This approach is well-documented, deterministic, and lets Claude write the entire structure programmatically.

**Primary recommendation:** Start with the official Figma remote MCP (`claude mcp add --transport http figma https://mcp.figma.com/mcp`). If it can create Figma components directly, use it. If it cannot (likely — it is primarily a read/code-gen tool), pivot to the Figma Plugin API bootstrap script as the automated creation path, with the user manually refining visual fidelity and running the human-verify checkpoint.

### Pattern 1: Figma Plugin API Bootstrap Script

**What:** A JavaScript file Claude writes that the user pastes into the Figma plugin console (or runs as a development plugin) to programmatically create the page structure, section header frames, and placeholder components.

**When to use:** When the official MCP cannot create components/pages directly — this is the primary fallback.

**Key API calls:**
```javascript
// Source: https://developers.figma.com/docs/plugins/api/properties/figma-createpage/
// Source: https://developers.figma.com/docs/plugins/api/properties/figma-createcomponent/
// Source: https://developers.figma.com/docs/plugins/api/properties/figma-combineasvariants/

// Pages
const tokensPage = figma.createPage();
tokensPage.name = 'Tokens';
await figma.setCurrentPageAsync(tokensPage);  // async setter required (dynamic page loading)

// Component
const btn = figma.createComponent();
btn.name = 'Buttons/Primary';  // slash convention creates Figma category grouping
btn.resize(120, 40);
btn.fills = [{ type: 'SOLID', color: { r: 0.063, g: 0.725, b: 0.506 } }]; // #10b981 emerald

// Auto-layout on a frame (must set layoutMode before other layout props)
const frame = figma.createFrame();
frame.layoutMode = 'HORIZONTAL';
frame.primaryAxisSizingMode = 'AUTO';    // shrink-wrap children horizontally
frame.counterAxisSizingMode = 'AUTO';   // shrink-wrap children vertically
frame.itemSpacing = 16;
frame.paddingLeft = 24;
frame.paddingRight = 24;

// Font must be loaded before setting text characters
await figma.loadFontAsync({ family: 'Lato', style: 'Regular' });
await figma.loadFontAsync({ family: 'Lato', style: 'Bold' });
const label = figma.createText();
label.characters = 'Buttons';
label.fontName = { family: 'Lato', style: 'Bold' };
label.fontSize = 16;

// Variants: create individual components first, then combine
const primaryBtn = figma.createComponent();
primaryBtn.name = 'State=Default';
const disabledBtn = figma.createComponent();
disabledBtn.name = 'State=Disabled';
const variantSet = figma.combineAsVariants([primaryBtn, disabledBtn], figma.currentPage);
variantSet.name = 'Buttons/Primary';
```

**Critical: `figma.setCurrentPageAsync()` is required (not `figma.currentPage =`)** — Figma introduced dynamic page loading and the synchronous setter was deprecated. Plugins must use the async version to ensure the page is loaded before operating on it.

### Pattern 2: Atomic Design Naming Convention

**What:** Figma's slash `/` in a component name creates visual grouping in the Assets panel. The CONTEXT.md decision uses `Category/ComponentName` (e.g., `Buttons/Primary`, `Inputs/Text`, `Navigation/Sidebar`).

**Variant naming:** Component sets (variants) use property=value syntax in the component name. For example, to create a button with state variants:
```
Component name on ComponentSetNode: "Buttons/Primary"
Individual variant names:           "State=Default", "State=Disabled", "State=Loading"
```
This produces a component set in the Assets panel under Buttons → Primary with a "State" property.

**Feature group naming (Phase 2):** `Exercise/Card`, `Exercise/SearchBar` etc. — the feature name replaces the atomic category. No "Phase 2" prefix in the component name.

### Pattern 3: Design Token Representation in Figma

**What:** The Tokens page uses colored rectangles (swatches) labeled with token name and hex value, arranged in rows by palette group. Typography is a text sample showing each weight/size combination.

**Figma does not have a native "token" type** — tokens are represented as fills on rectangles (for color) and text nodes (for type). The `docs/design-tokens.md` file is the source of truth; Figma is a visual reference.

**Token values from `globals.css`:**
```
--color-bg-page:      #0f172a   (dark navy — page background)
--color-bg-surface:   #1e293b   (lighter navy — card/panel surface)
--color-border:       #334155   (slate border)
--color-accent:       #10b981   (emerald green — primary CTA)
--color-accent-hover: #34d399   (lighter emerald — hover)
--color-text-primary: #e2e8f0   (near-white — body text)
--color-error:        #ef4444   (red — error state)
--color-error-light:  #f87171   (lighter red — error text)
```
Typography: Lato, weights 400 (Regular) and 700 (Bold). Scale is Tailwind's default text scale (xs through 2xl).

### Anti-Patterns to Avoid

- **Flat frames instead of components:** Never create non-component frames for the library elements. Flat frames cannot be instanced and break the "update once, reflects everywhere" principle for future sub-phase use.
- **Synchronous `figma.currentPage =` assignment in plugin scripts:** Deprecated since Figma introduced dynamic page loading. Use `await figma.setCurrentPageAsync(page)`.
- **Empty `combineAsVariants` call:** `figma.combineAsVariants([])` will throw. Must pass at least one component node.
- **Modifying text without `loadFontAsync`:** Any plugin that sets `.characters`, `.fontSize`, or `.fontName` without awaiting `figma.loadFontAsync()` first will throw a runtime error.
- **Using "First Draft" / "Make Designs" AI on free plan:** These features require a paid Professional plan. They will not be available on the free Starter plan.

## Don't Hand-Roll

| Problem | Don't Build | Use Instead | Why |
|---------|-------------|-------------|-----|
| Color-to-Figma RGB conversion | Manual hex → `{r,g,b}` parsing logic | Inline calculation: `parseInt(hex.slice(1,3),16)/255` per channel | Trivial but error-prone; just do it inline in the bootstrap script |
| Page creation and navigation | Custom async wrapper | `figma.createPage()` + `await figma.setCurrentPageAsync()` | Plugin API handles the async contract cleanly |
| Variant grouping | Manually naming components with prefix conventions | `figma.combineAsVariants([c1, c2], parent)` | Produces a native ComponentSetNode with Figma's built-in variant UI |
| Token documentation | Manual markdown table | Claude-generated `docs/design-tokens.md` from `globals.css` | Parsing `@theme inline` block is mechanical; automation guarantees accuracy |

**Key insight:** The Figma Plugin API is the single most powerful automated path available on the free plan. It runs inside Figma, needs no external server, and exposes the complete document model. The bootstrap script should do as much structural work as possible, leaving only visual polish for the user's human-verify pass.

## Common Pitfalls

### Pitfall 1: Official Figma MCP is read + code-gen, not a component creator

**What goes wrong:** Expecting the official Figma remote MCP (`https://mcp.figma.com/mcp`) to create Figma components and pages. The MCP's write capability is limited to capturing live web UI into Figma layers, not authoring Figma design components from scratch via Claude prompts.

**Why it happens:** The MCP's marketing language ("generate designs from live UI") sounds like Claude can author Figma files. In practice, it converts rendered HTML/CSS to Figma layers — it is not a Figma plugin with full document write access.

**How to avoid:** Set up the official MCP first (one command, zero friction, free plan compatible). Test it explicitly: ask it to create a page and a component. If it cannot, pivot immediately to the Plugin API bootstrap script.

**Warning signs:** Official MCP responds with "I can extract design context" or "point me at a Figma link to generate code" rather than creating nodes in the current Figma file.

### Pitfall 2: `claude-talk-to-figma-mcp` requires Figma desktop app

**What goes wrong:** Trying to use `claude-talk-to-figma-mcp` with Figma running in the browser (figma.com). The WebSocket bridge only connects to the Figma desktop app.

**Why it happens:** The plugin runs inside the Figma desktop app and communicates with the local WebSocket server. Browser-based Figma does not support the same plugin/WebSocket bridge.

**How to avoid:** If attempting `claude-talk-to-figma-mcp`, confirm the user has Figma desktop installed. If not, do not use this path — fall back to the Plugin API bootstrap script instead.

**Warning signs:** The Figma plugin "Connect" button shows no channel, or the WebSocket server (`bun run socket` / `npx claude-talk-to-figma-mcp`) connects but Figma plugin is not visible in the desktop app.

### Pitfall 3: Plugin script fails silently on page operations

**What goes wrong:** The bootstrap script creates pages and components, but subsequent operations on those pages fail because `setCurrentPageAsync` was not awaited or the page was not fully loaded before nodes were appended.

**Why it happens:** Figma introduced dynamic page loading — pages are lazy-loaded. Any operation on a page's `.children` before the page is loaded produces a runtime error or silently no-ops.

**How to avoid:** Every page switch in the plugin must use `await figma.setCurrentPageAsync(page)` before any read or write operation on that page. Structure the bootstrap script as a top-level `async` function.

**Warning signs:** Components appear on the wrong page, or node.appendChild() throws "Cannot read children of unloaded page."

### Pitfall 4: Variant naming mismatch breaks the Assets panel grouping

**What goes wrong:** Component names use inconsistent slash depth (e.g., `Buttons/Primary/Default` vs `Buttons/Primary` as the ComponentSetNode name) causing the Assets panel to show duplicated or nested groups.

**Why it happens:** In Figma, the `ComponentSetNode.name` is the group path (e.g., `Buttons/Primary`), while individual variant component names should be only the property values (e.g., `State=Default`), not full paths.

**How to avoid:** When using `combineAsVariants`, set the name on the returned `ComponentSetNode` (the group), not on the child `ComponentNode` variants. Individual variant names should be `Property=Value` format only.

**Warning signs:** Assets panel shows deeply nested folders, or component instances show confusing names in the properties panel.

### Pitfall 5: Free plan file confusion — drafts vs. team project files

**What goes wrong:** Believing the free plan allows only 3 draft files. A previous CONTEXT.md note said "free tier — 1 of 3 drafts" but the current Figma Starter plan (as of 2025-2026) allows unlimited personal drafts. The 3-file limit applies only to files within a team project.

**Why it happens:** Figma changed its plan structure in October 2024-2025: personal drafts moved into teams but remain unlimited. The prior limit was for team project files.

**How to avoid:** Create the Forge UI Library file in personal Drafts (not inside a team project). This is unlimited on the free plan. The file is still shareable via link.

**Warning signs:** Getting a "file limit reached" error means the file is being created inside a team project space — move it to Drafts.

### Bootstrap Script Skeleton (Figma Plugin API)

```javascript
// Source: https://developers.figma.com/docs/plugins/api/properties/figma-createpage/
// Source: https://developers.figma.com/docs/plugins/api/properties/figma-createcomponent/
// Source: https://developers.figma.com/docs/plugins/async-tasks/

(async () => {
  // Load fonts before any text manipulation
  await figma.loadFontAsync({ family: 'Lato', style: 'Regular' });
  await figma.loadFontAsync({ family: 'Lato', style: 'Bold' });

  // Remove default "Page 1" if present
  if (figma.root.children.length === 1 && figma.root.children[0].name === 'Page 1') {
    // Do NOT remove the only page — Figma requires at least 1 page
    // Rename it instead, then create the rest
    figma.root.children[0].name = 'Tokens';
  }

  // Create pages in order
  const pageNames = ['Atoms', 'Molecules', 'Organisms', 'Phase 2: Exercise Library'];
  for (const name of pageNames) {
    const p = figma.createPage();
    p.name = name;
  }

  // Switch to Tokens page and build token swatches
  const tokensPage = figma.root.children.find(p => p.name === 'Tokens');
  await figma.setCurrentPageAsync(tokensPage);

  // Color swatch helper
  function createSwatch(name, hex, x, y) {
    const rect = figma.createRectangle();
    rect.resize(80, 80);
    rect.x = x;
    rect.y = y;
    const r = parseInt(hex.slice(1, 3), 16) / 255;
    const g = parseInt(hex.slice(3, 5), 16) / 255;
    const b = parseInt(hex.slice(5, 7), 16) / 255;
    rect.fills = [{ type: 'SOLID', color: { r, g, b } }];
    const label = figma.createText();
    label.characters = name + '\n' + hex;
    label.fontSize = 10;
    label.x = x;
    label.y = y + 88;
    return [rect, label];
  }

  // Section header helper
  function createSectionHeader(text, x, y) {
    const label = figma.createText();
    label.characters = text;
    label.fontName = { family: 'Lato', style: 'Bold' };
    label.fontSize = 16;
    label.fills = [{ type: 'SOLID', color: { r: 0.886, g: 0.906, b: 0.941 } }]; // #e2e8f0
    label.x = x;
    label.y = y;
    return label;
  }

  // ... populate swatches and components per page
  figma.closePlugin('Forge UI Library structure created.');
})();
```

### Component + Variants Pattern

```javascript
// Source: https://developers.figma.com/docs/plugins/api/properties/figma-combineasvariants/
// Source: https://forum.figma.com/ask-the-community-7/combineasvariants-to-create-a-new-component-set-31209/

// Create individual state variants first
function makeButtonVariant(stateName, fillHex, opacity = 1) {
  const c = figma.createComponent();
  c.name = `State=${stateName}`;  // variant property=value syntax
  c.resize(120, 40);
  const r = parseInt(fillHex.slice(1, 3), 16) / 255;
  const g = parseInt(fillHex.slice(3, 5), 16) / 255;
  const b = parseInt(fillHex.slice(5, 7), 16) / 255;
  c.fills = [{ type: 'SOLID', color: { r, g, b }, opacity }];
  return c;
}

const btnDefault  = makeButtonVariant('Default',  '#10b981');
const btnDisabled = makeButtonVariant('Disabled', '#10b981', 0.4);
const btnLoading  = makeButtonVariant('Loading',  '#10b981', 0.7);

// combineAsVariants creates a ComponentSetNode — name IT, not the children
const btnSet = figma.combineAsVariants([btnDefault, btnDisabled, btnLoading], figma.currentPage);
btnSet.name = 'Buttons/Primary';  // slash = Assets panel category grouping
```
