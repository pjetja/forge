    # Figma Workflow — Forge UI Library

    ## Figma File
    ```
    Link: FIGMA_LINK_PLACEHOLDER
    ```
    (This placeholder is replaced by the user after creating the Figma file in Plan 02.)

    ## Purpose
    A single Figma file that serves as the component library and design canvas for all future phases. Build screens in Figma first, then code. This avoids mid-implementation design decisions.

    ## Library Structure (5 pages)
    List the 5 pages per CONTEXT.md: Tokens, Atoms, Molecules, Organisms, Phase 2: Exercise Library. For each page, list what components live there (matching CONTEXT.md component scope exactly).

    ## How to Use for a New Phase
    Step-by-step instructions for each UI sub-phase (02.1, 03.1, etc.):
    1. Open the Figma file
    2. Create a new page named "Phase X.Y: [Feature Name]" for screen compositions
    3. Use component instances from the library pages (never modify main components)
    4. Design screens showing all states (empty, loading, data, error)
    5. Get human approval via /gsd:discuss-phase before coding
    6. During coding, reference the Figma screen as the design spec

    ## Component Naming Convention
    - Slash grouping: `Category/ComponentName` (e.g. `Buttons/Primary`, `Inputs/Text`)
    - Variant names: `Property=Value` format (e.g. `State=Default`, `State=Disabled`)
    - Phase 2 feature group: `Exercise/Card`, `Exercise/SearchBar` (no phase prefix in name)

    ## When to Update Tokens
    Update docs/design-tokens.md AND Figma token swatches if src/app/globals.css @theme inline changes. Manual sync — no automated token bridge.

    ## Bootstrap Script
    The file `docs/figma-bootstrap.js` creates the base page structure programmatically. Run it via Figma's built-in Plugin console (Plugins → Development → Open Console) or as a development plugin. See Plan 02 for instructions.

    **B. Create docs/figma-bootstrap.js**

    A complete, runnable Figma Plugin API script that creates the full Forge UI Library structure. The script must:

    1. Be wrapped in a top-level `(async () => { ... })()` — required for `await` at top level
    2. Load Lato fonts before any text operations: `await figma.loadFontAsync({ family: 'Lato', style: 'Regular' })` and `await figma.loadFontAsync({ family: 'Lato', style: 'Bold' })`
    3. Rename the default "Page 1" to "Tokens" (never remove the only page — Figma requires at least 1)
    4. Create the remaining 4 pages: "Atoms", "Molecules", "Organisms", "Phase 2: Exercise Library"
    5. Switch to each page using `await figma.setCurrentPageAsync(page)` — NOT `figma.currentPage = page` (deprecated)

    **Tokens page:** Build color swatches for all 8 design tokens. Use inline hex-to-RGB conversion: `parseInt(hex.slice(1,3),16)/255`. Each swatch: 80x80 rectangle with SOLID fill + label text below (token name + hex value). Arrange in 2 rows of 4, 120px spacing. Add section header "Colors" (Lato Bold 20px, #e2e8f0). Typography section to the right (x=600): show "Lato Regular / Bold" at each Tailwind scale step (xs→2xl) as text nodes.

    **Atoms page:** Create component sets using `figma.combineAsVariants()`. Build these component sets:

    - `Buttons/Primary`: variants State=Default (#10b981 fill), State=Hover (#34d399 fill), State=Disabled (#10b981 fill at 0.4 opacity), State=Loading (#10b981 fill at 0.7 opacity). Size: 120x40 each.
    - `Buttons/Secondary`: variants State=Default (transparent fill, #334155 stroke 1px), State=Hover (#1e293b fill, #10b981 stroke), State=Disabled (transparent, #334155 stroke at 0.4 opacity). Size: 120x40.
    - `Buttons/Ghost`: variants State=Default (transparent fill, no stroke), State=Hover (#1e293b fill). Size: 120x40.
    - `Inputs/Text`: variants State=Default (#1e293b fill, #334155 stroke), State=Focus (#1e293b fill, #10b981 stroke), State=Error (#1e293b fill, #ef4444 stroke), State=Disabled (#1e293b fill at 0.5 opacity). Size: 280x44.
    - `Inputs/Textarea`: same 4 states as Inputs/Text. Size: 280x100.
    - `Badges/Tag`: variants Type=Accent (#10b981 fill, 8px corner radius), Type=Muted (#334155 fill). Size: 80x24.
    - `Logo/Horizontal`: single component (no variants), 120x28 rectangle with accent fill placeholder (user replaces with actual SVG). Name: `Logo/Horizontal`.
    - `Logo/Icon`: single component, 28x28 with accent fill placeholder. Name: `Logo/Icon`.

    Place each component set on the Atoms page with a Lato Bold 16px section header label above it. Space sections 80px apart vertically. Start at x=40, y=80.

    **Molecules page:** Create these component sets:
