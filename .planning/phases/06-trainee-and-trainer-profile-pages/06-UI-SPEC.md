## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, stat dot separators (`·`), inline padding |
| sm | 8px | Compact element spacing, input label-to-field gap |
| md | 16px | Default element spacing, card inner padding, field row gap |
| lg | 24px | Section padding, between form sections |
| xl | 32px | Layout gaps between major page sections |
| 2xl | 48px | Major section breaks (e.g. avatar header to form body) |
| 3xl | 64px | Page-level top/bottom padding |

Exceptions:
- Avatar in NavHeader: 40px rendered size (matches existing `h-10 w-10` pattern)
- Avatar on profile pages and trainee detail header: 80px rendered size (prominent display context)
- Touch targets (avatar button, save buttons): minimum 44px height to meet mobile tap target requirement

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | 400 | 1.5 |
| Label | 14px | 400 | 1.4 |
| Heading | 20px | 700 | 1.2 |
| Display | 28px | 700 | 1.2 |

Notes:
- Body (16px / 400 / 1.5): form field values, bio text, goals text, trainer notes textarea, read-only stats
- Label (14px / 400 / 1.4): form field labels, section sub-labels, compliance stat lines ("Last workout: Mar 14"), physical stats chip row text, helper text (e.g. avatar Gravatar hint, trainer notes privacy indicator)
- Heading (20px / 700 / 1.2): profile page section headings ("Profile", "My Trainer"), trainee detail section headings
- Display (28px / 700 / 1.2): profile page primary name display (above avatar area)

Source: Matches existing app typography conventions observed in trainer/trainee pages.

## Interaction Contracts

### Avatar in NavHeader

- **Idle state:** 40×40px `rounded-full` image with `bg-accent/20` background (shows while Gravatar loads — Gravatar `d=mp` fallback ensures image always resolves)
- **Hover state:** `ring-2 ring-accent ring-offset-2 ring-offset-bg-page` — 2px emerald ring with dark offset
- **Click:** Next.js `Link` navigation to `/trainer/profile` or `/trainee/profile`; no dropdown
- **Sign-out:** Removed from header; accessible from the profile page as a standalone button at the bottom of the page

Decision (Claude's discretion): Sign-out goes on the profile page, not in a dropdown. A dropdown adds interaction complexity and mobile tap precision concerns. Profile page is a natural home for account actions.

### Profile Form Save

- **Idle:** Form fields pre-populated from DB values; Save button is accent-colored, full-width on mobile, right-aligned on desktop
- **Submitting:** Save button shows "Saving..." text and `opacity-60 cursor-not-allowed` state; fields remain editable
- **Success:** Inline confirmation text below button: "Changes saved." in `text-accent text-sm`; auto-clears after 3 seconds
- **Error:** Inline error text below button: error message from Server Action in `text-error text-sm`
- **Validation:** Client-side: name field must not be empty (disabled Save button if blank); all other fields optional

### Trainer Notes Editor (on trainee detail page)

- **Layout:** Full-width textarea (min 4 rows) with `bg-bg-surface border-border` styling; label "Trainer notes (private)" in 14px / 400
- **Save trigger:** Explicit "Save notes" button below the textarea — no autosave (avoids accidental saves, aligns with existing explicit-save pattern in the app)
- **Idle with no notes:** Textarea placeholder "Add private notes about this trainee..."
- **Saving/success/error:** Same inline feedback pattern as profile form
- **Privacy indicator:** Small helper text below label: "Only visible to you" in 14px / 400 `text-text-primary opacity-50`

Decision (Claude's discretion): Explicit Save button over autosave. Autosave creates ambiguity about whether a half-typed note was persisted. The app already uses explicit saves everywhere.

### Trainee Goals on Trainer Detail Page

- **Layout:** Read-only section titled "Trainee goals" (20px / 700); content as paragraph text (16px / 400)
- **Empty state:** "No goals set." in `text-text-primary opacity-50`
- **Not editable:** No pencil/edit icon; explicitly read-only by design

### Physical Stats Chip Row

- **Format:** `{height_cm} cm · {weight_kg} kg · Age {computed_age}` — single line of 14px / 400 text at `opacity-70`
- **Partial data:** Only render present values separated by `·`; omit blank fields entirely
- **All empty:** Do not render the row at all; no "—" placeholder for the whole row

### My Trainer Card (on trainee profile page)

Decision (Claude's discretion): "My Trainer" is a section within `/trainee/profile`, not a separate tab. The trainee profile page has limited content; a tab is unnecessary overhead for a single read-only card.

- **Layout:** Card with `bg-bg-surface` background, `rounded-lg`, `p-6`; avatar (40px) left-aligned beside name and email; bio below in body text
- **Conditional render:** Only shown if a trainer connection exists
- **No connection state:** Section not rendered; no empty state message (no trainer means trainee signed up solo — don't prompt action in this phase)

### Compliance Stats on Trainer Home Cards

- **Position:** Two lines below the existing plan status text within each trainee card
- **Format line 1:** "Last workout: {Day Month}" (e.g. "Last workout: 14 Mar") — 14px / 400 / `opacity-50`
- **Format line 2:** "{N} this week" (e.g. "2 this week") — 14px / 400 / `opacity-50`; renders on the same line as last workout with ` · ` separator to keep compact
- **No sessions state:** Single line "No sessions yet" — 14px / 400 / `opacity-50`

## Page Structure

### `/trainer/profile`

Primary focal point: avatar + display name block at the top of the page — this is the identity anchor for the entire profile page.

```
[Avatar 80px — ring-accent] [Display name — 28px/700]
                             [email — 14px/400/opacity-50]
                             [helper: "Set your avatar at gravatar.com" — 14px/opacity-50]

[Section: Profile]
  [Name field — editable]
  [Bio field — editable textarea, 4 rows]
  [Save changes button — accent]
  [Inline success/error feedback]

[Sign out button — ghost/destructive-text, bottom of page]
```

### `/trainee/profile`

Primary focal point: avatar + display name block at the top of the page — this is the identity anchor for the entire profile page.

```
[Avatar 80px — ring-accent] [Display name — 28px/700]
                             [email — 14px/400/opacity-50]
                             [helper: "Set your avatar at gravatar.com" — 14px/opacity-50]

[Section: Profile]
  [Name field — editable]
  [Goals textarea — editable, 4 rows]
  [Height field — editable, number input, cm]
  [Weight field — editable, number input, kg]
  [Date of birth field — editable, date input]
  [Save changes button — accent]
  [Inline success/error feedback]

[Section: My Trainer] (only if connection exists)
  [Card: trainer avatar (40px) + name + email + bio]

[Sign out button — ghost/destructive-text, bottom of page]
```
