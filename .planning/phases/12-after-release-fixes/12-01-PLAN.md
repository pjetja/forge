<objective>
Add configurable parameters to progression modes:
- RPE mode → `rpe_target` integer (1–10)
- RIR mode → `rir_target` integer (0–5)
- Linear mode → `weight_increment_per_week` numeric (kg/week)

Trainer fills in the value field when they select a mode. Trainee sees the configured target during their workout session.

Also fix the Drizzle schema.ts gap: `progressionMode` and `tempo` exist in DB (migration 0006) but are missing from the TypeScript type definitions.
</objective>

<tasks>

</read_first>

**File:** `src/lib/db/migrations/0012_progression_targets.sql` (new)

Add three new nullable columns to both tables:

```sql
ALTER TABLE schema_exercises
  ADD COLUMN rpe_target INTEGER CHECK (rpe_target BETWEEN 1 AND 10),
  ADD COLUMN rir_target INTEGER CHECK (rir_target BETWEEN 0 AND 5),
  ADD COLUMN weight_increment_per_week NUMERIC(5, 2);

ALTER TABLE assigned_schema_exercises
  ADD COLUMN rpe_target INTEGER CHECK (rpe_target BETWEEN 1 AND 10),
  ADD COLUMN rir_target INTEGER CHECK (rir_target BETWEEN 0 AND 5),
  ADD COLUMN weight_increment_per_week NUMERIC(5, 2);
```

Then replace the `assign_plan()` RPC to copy the three new columns (plus already-existing `tempo`, `progression_mode`). Follow the existing pattern in 0007 — copy the full function body, extend the SELECT to include the new columns, and propagate from `p_weight_overrides` JSONB with COALESCE.

**File:** `src/lib/db/schema.ts`

Fix the existing type gap: add `progressionMode` and `tempo` to both `schemaExercises` and `assignedSchemaExercises` table definitions. Then add the three new fields.

