<objective>
Two improvements in this plan:

1. **Body Weight tab always visible** — On the trainer's trainee-detail page, the "Body Weight" tab currently only appears in the tab bar when body-weight access has been approved. Requesting access UI is shown inline somewhere else. Fix: always show the "Body Weight" tab; put the request/pending/declined UI inside the tab.

2. **i18n audit** — Scan all 8 translation JSON files for obviously wrong Polish translations, missing keys, or keys with English values in the Polish locale. Fix identified issues.
   </objective>

<tasks>

## Task 1 — Body Weight tab always visible

**File:** `src/app/(trainer)/trainer/trainees/[traineeId]/page.tsx`

Current state: `...(bodyWeightAccess === 'approved' ? [{ key: 'body-weight', ... }] : [])` in TabSwitcher.

Change to: always include `{ key: 'body-weight', label: t('traineeDetail.tabs.bodyWeight') }` in the tabs array (remove the conditional).

In the `{activeTab === 'body-weight' && ...}` render block, update the logic:

- `bodyWeightAccess === null` → render `<RequestBodyWeightAccessButton ... />` with a brief explanation ("Request access to view body weight data")
- `bodyWeightAccess === 'pending'` → render a "Request sent — awaiting trainee approval" message
- `bodyWeightAccess === 'declined'` → render a "Request declined" message with an option to re-request
- `bodyWeightAccess === 'approved'` → render `<BodyWeightTab ... />` (existing)

Move `<RequestBodyWeightAccessButton>` out of wherever it currently lives (inline in another tab or the page header) and put it inside the body-weight tab. Remove the inline render from its original location.

Ensure the `activeTab` defaulting logic includes 'body-weight' as a valid tab: update the valid tabs array in the page: `['exercises', 'goals', 'notes', 'body-weight']` (already includes it — confirm no change needed here).

## Task 2 — i18n audit

Read all 8 translation files:

- `messages/en/auth.json`, `messages/pl/auth.json`
- `messages/en/common.json`, `messages/pl/common.json`
- `messages/en/trainee.json`, `messages/pl/trainee.json`
- `messages/en/trainer.json`, `messages/pl/trainer.json`

For each PL file, check:

1. Any key with an English-language value (copy-paste not translated)
2. Any key present in EN but missing from PL (or present in PL but missing in EN)
3. Any obviously garbled or clearly wrong translation

Fix all identified issues. Document each fix in the plan SUMMARY with: key path, old value, new value.

Note: This is a best-effort audit. Native-level Polish review is not required — catch obvious systemic issues (untranslated blocks, structural mismatches).

</tasks>
