<task type="auto">
  <name>Task 2: Create .github/workflows/ci.yml</name>
  <files>.github/workflows/ci.yml</files>
  <action>
    Create `.github/workflows/ci.yml` with this exact content. Uses `pnpm/action-setup` for pnpm, pins to Node 20 (matches local dev), and caches the pnpm store for fast runs:

    ```yaml
    name: CI

    on:
      push:
        branches: [main]
      pull_request:
        branches: [main]

    jobs:
      build:
        name: Build
        runs-on: ubuntu-latest

        steps:
          - name: Checkout
            uses: actions/checkout@v4

          - name: Install pnpm
            uses: pnpm/action-setup@v3
            with:
              version: 9

          - name: Set up Node.js
            uses: actions/setup-node@v4
            with:
              node-version: 20
              cache: pnpm

          - name: Install dependencies
            run: pnpm install --frozen-lockfile

          - name: Build
            run: pnpm build
            env:
              # Provide stub env vars so Next.js build succeeds in CI
              # (no real Supabase connection needed for build-time checks)
              NEXT_PUBLIC_SUPABASE_URL: https://placeholder.supabase.co
              NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY: placeholder-anon-key
              SUPABASE_SERVICE_ROLE_KEY: placeholder-service-role-key
              NEXT_PUBLIC_SITE_URL: https://placeholder.vercel.app
    ```

    After creating the file, commit and push it:
    ```bash
    git add .github/workflows/ci.yml
    git commit -m "ci: add GitHub Actions build workflow"
    git push
    ```

  </action>
