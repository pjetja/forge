<objective>
Import the GitHub repo into Vercel, configure all 4 required environment variables, trigger the first production deployment, and confirm the live URL works.

Purpose: The app needs a public HTTPS URL for beta testing. Vercel provides zero-config Next.js hosting with automatic deployments on every push to main. Env vars — especially NEXT_PUBLIC_SITE_URL — must be set before the first deploy, or signup confirmation emails will redirect to localhost.

Output: Live app at `https://[your-app].vercel.app`; all env vars set in Vercel; landing page + login verified working.
</objective>

<tasks>

<task type="human_action">
  <name>Task 1: Import GitHub repo into Vercel</name>
  <what-needed>
    1. Go to https://vercel.com/new
    2. Click "Import Git Repository"
    3. Connect your GitHub account if not already connected
    4. Select the `forge` repository
    5. Vercel will auto-detect **Next.js** — confirm the framework preset is "Next.js"
    6. **IMPORTANT — Do NOT click Deploy yet.** First configure env vars in Task 2.
    7. Note the project slug Vercel suggests (e.g. `forge` or `forge-[suffix]`) — this becomes your `[your-app]` in `https://[your-app].vercel.app`
  </what-needed>
  <resume-signal>Type "imported" once you can see the Vercel project config screen (before deploy)</resume-signal>
</task>

<task type="human_action">
  <name>Task 2: Add all environment variables in Vercel dashboard</name>
  <what-needed>
    On the Vercel project config screen (or Settings → Environment Variables after import):

    Add the following 4 env vars. Get the values from your local `.env.local` file:

    | Variable | Value | Sensitive? |
    |----------|-------|-----------|
    | `NEXT_PUBLIC_SUPABASE_URL` | Your Supabase project URL | No |
    | `NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY` | Your Supabase anon key | No |
    | `SUPABASE_SERVICE_ROLE_KEY` | Your Supabase service_role key | **YES — mark as Sensitive** |
    | `NEXT_PUBLIC_SITE_URL` | `https://[your-app].vercel.app` (use the URL from Task 1) | No |

    **For each variable:**
    - Set Environment: ✅ Production, ✅ Preview, ✅ Development
    - For `SUPABASE_SERVICE_ROLE_KEY`: click the lock/sensitive toggle

    **Do NOT add `DATABASE_URL`** — the app does not use it at runtime (Drizzle is schema-only).

    After all 4 are added, click **Deploy**.

  </what-needed>
  <resume-signal>Type "deploying" once you've clicked Deploy</resume-signal>
</task>

<task type="human_action">
  <name>Task 3: Wait for first deployment to complete</name>
  <what-needed>
    1. Watch the Vercel build log in the dashboard
    2. The build runs `pnpm build` (same as your local build — should pass since Plan 11-01 confirmed it)
    3. Wait for "Deployment complete" / green checkmark
    4. Note your live URL: `https://[your-app].vercel.app`

    If the build fails:
    - Check the error in the Vercel build log
    - Most common cause: missing env var — verify all 4 are set
    - A TypeScript error: run `pnpm build` locally to reproduce and fix

  </what-needed>
  <resume-signal>Type "deployed" plus your live URL (e.g. "deployed https://forge-abc123.vercel.app")</resume-signal>
</task>

<task type="auto">
  <name>Task 4: Update .env.local.example with Vercel deployment note</name>
  <files>.env.local.example</files>
  <action>
    Update `.env.local.example` to add a comment clarifying what NEXT_PUBLIC_SITE_URL should be in production. Find the `NEXT_PUBLIC_SITE_URL` line added in Plan 11-01 and update its comment to include the Vercel note:

    Replace:
    ```
    # Site URL — set to http://localhost:3000 locally; set to https://[your-app].vercel.app in Vercel dashboard

    NEXT_PUBLIC_SITE_URL=http://localhost:3000
    ```

    Then commit this change:
    ```bash
    git add .env.local.example
    git commit -m "docs: clarify NEXT_PUBLIC_SITE_URL usage for Vercel deployment"
    git push
    ```
