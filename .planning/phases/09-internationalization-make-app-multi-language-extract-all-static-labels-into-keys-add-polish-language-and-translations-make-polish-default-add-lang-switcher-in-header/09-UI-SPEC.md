## Component Contract: LanguageSwitcher

This is the only new UI component in Phase 9. All other changes are text substitution (no visual change to existing components beyond string replacements).

**Placement:**
- Desktop header row 1 (`h-14 px-4 flex items-center justify-between`): insert before the avatar in the `hidden md:flex items-center gap-3` container.
- Mobile sidebar bottom section: insert as a row above the Profile link in the `px-4 py-4 border-t border-border space-y-3` footer block.

**Desktop render:**
```
PL | EN
```
- `PL` and `EN` are `<button>` elements (not links — locale switch does not change URL per D-04)
- Separator `|` is a `<span aria-hidden="true">` with `text-border mx-1`
- Active locale button: `text-accent font-bold text-sm cursor-default`
- Inactive locale button: `text-text-primary/50 hover:text-accent font-bold text-sm transition-colors cursor-pointer`
- Container: `<div class="flex items-center text-sm">` — no border, no background, no box

**Mobile sidebar render:**
- Same visual logic as desktop
- Minimum touch target: `min-h-[44px]` on each button to satisfy gym-use ergonomics
- Display as `flex items-center gap-1` row, left-aligned within the sidebar footer

**Interaction:**
- Click inactive locale button → set `NEXT_LOCALE` cookie → trigger page reload with new locale
- Active locale button does nothing (cursor-default, not cursor-pointer)
- No confirmation dialog — language switch is instantly reversible

**Accessibility:**
- Each button has `aria-label="Switch to English"` / `aria-label="Zmien jezyk na angielski"` (translated)
- Active button has `aria-pressed="true"`; inactive has `aria-pressed="false"`
- Container has `role="group"` and `aria-label="Language"` / `aria-label="Jezyk"`

## Copywriting Contract

Phase 9 is a translation/extraction phase. The copywriting contract governs the **English** reference strings (source language for `messages/en/`) and the key naming convention. Polish translations mirror these semantically.

| Element | English Copy | Polish Copy |
|---------|-------------|-------------|
| Language switcher active label (PL) | — | PL |
| Language switcher active label (EN) | EN | — |
| Language switcher aria-label (switch to EN) | "Switch to English" | "Switch to English" |
| Language switcher aria-label (switch to PL) | "Zmien jezyk na polski" | "Zmien jezyk na polski" |
| Language group aria-label | "Language" | "Jezyk" |
| Primary CTA (language switch) | none — immediate action, no confirmation | none |
| Empty state heading | "Nothing here yet" | "Brak danych" |
| Empty state body | "Check back after completing a session." | "Sprawdz ponownie po ukonczeniu treningu." |
| Error state | "Something went wrong. Please refresh the page." | "Cos poszlo nie tak. Odswiez strone." |
| Destructive confirmation | none in this phase | none in this phase |

**Key naming convention** (Claude's discretion — established here as contract):
- Dot-separated, feature-scoped: `{namespace}.{element}.{variant}`
- Examples:
  - `common.button.save` → "Save" / "Zapisz"
  - `common.button.cancel` → "Cancel" / "Anuluj"
  - `common.empty.heading` → "Nothing here yet" / "Brak danych"
  - `nav.trainees` → "Trainees" / "Kursanci"
  - `nav.plans` → "Plans" / "Plany"
  - `nav.exerciseLibrary` → "Exercise Library" / "Biblioteka cwiczen"
  - `nav.help` → "Help" / "Pomoc"
  - `nav.language` → "Language" / "Jezyk"
  - `auth.login.heading` → "Sign in to your account" / "Zaloguj sie"
  - `trainer.trainees.empty` → "No trainees yet." / "Brak kursantow."
  - `trainee.plans.empty` → "No plan assigned yet." / "Brak przypisanego planu."

**Namespace file mapping:**
- `messages/{locale}/common.json` — shared: buttons, validation, empty states, toasts, nav
- `messages/{locale}/auth.json` — login, signup, forgot-password, reset-password
- `messages/{locale}/trainer.json` — trainer-specific UI
- `messages/{locale}/trainee.json` — trainee-specific UI

## Phase-Specific Interaction Notes

### What changes visually in this phase

1. **LanguageSwitcher component** — new UI element in both nav headers (see Component Contract above)
2. **`lang` attribute on `<html>`** — updated dynamically to match active locale (`lang="pl"` or `lang="en"`)
3. **All visible text strings** — replaced with translated equivalents; no layout or spacing changes

### What does NOT change visually

- Page layouts, padding, component structures
- Color tokens, font, icon set
- Navigation structure (URLs unchanged per D-04)
- Any existing component beyond string replacement

### Locale persistence behavior

- First visit (no cookie): Polish rendered — no flash of English
- Language switch: cookie set, page reloads — no SPA transition animation needed
- Cookie name: next-intl default (`NEXT_LOCALE`) — expiry follows next-intl default (1 year)

### `lang` attribute on root layout

- `src/app/layout.tsx` currently has `lang="en"` hardcoded
- After Phase 9: `lang` is read from next-intl locale context and set dynamically
- Values: `"pl"` or `"en"` — standard BCP 47 language tags
