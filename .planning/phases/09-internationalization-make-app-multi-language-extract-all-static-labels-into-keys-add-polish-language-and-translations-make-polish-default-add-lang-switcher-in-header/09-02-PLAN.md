For each auth page and the shared SignupForm component:

1. Read the file and identify ALL user-visible strings: headings, labels, button text, placeholder text, error messages, link text, helper text.

2. For server components (page.tsx files), use `getTranslations`:
   ```typescript
   import { getTranslations } from 'next-intl/server';
   // Inside async function:
   const t = await getTranslations('auth');
   ```

3. For client components (LoginForm.tsx, SignupForm.tsx), use `useTranslations`:
   ```typescript
   import { useTranslations } from 'next-intl';
   // Inside component:
   const t = useTranslations('auth');
   ```

4. Replace each hardcoded string with `t('key.path')`. Use the key naming convention:
   - `auth.login.heading` for login page heading
   - `auth.login.emailLabel` for email field label
   - `auth.login.passwordLabel` for password field label
   - `auth.login.submit` for submit button text
   - `auth.login.noAccount` for "Don't have an account?" text
   - `auth.login.signUpLink` for sign up link text
   - `auth.signup.heading` for signup page heading
   - `auth.signup.trainerHeading` / `auth.signup.traineeHeading` for role-specific headings
   - `auth.signup.nameLabel`, `auth.signup.emailLabel`, `auth.signup.passwordLabel`, `auth.signup.confirmPasswordLabel`
   - `auth.signup.submit`
   - `auth.signup.hasAccount` for "Already have an account?" text
   - `auth.signup.loginLink` for login link text
   - `auth.forgotPassword.heading`, `auth.forgotPassword.submit`, `auth.forgotPassword.backToLogin`
   - `auth.resetPassword.heading`, `auth.resetPassword.newPasswordLabel`, `auth.resetPassword.confirmLabel`, `auth.resetPassword.submit`
   - `auth.verifyEmail.heading`, `auth.verifyEmail.body`, `auth.verifyEmail.resend`
   - Error messages: `auth.error.invalidCredentials`, `auth.error.emailTaken`, `auth.error.passwordMismatch`, etc.

5. For strings that are shared across auth pages (like "Email", "Password"), use `common` namespace via a second `const tc = useTranslations('common');` or re-use `t` with the `common.` prefix depending on what makes sense. Prefer keeping auth-specific strings in `auth` namespace.

6. Populate `messages/pl/auth.json` with ALL extracted keys and Polish translations.
   Polish translations should be natural, not machine-translated. Key examples:
   - "Sign in to your account" -> "Zaloguj sie do konta"
   - "Don't have an account?" -> "Nie masz konta?"
   - "Sign up" -> "Zarejestruj sie"
   - "Forgot password?" -> "Nie pamietasz hasla?"
   - "Send reset link" -> "Wyslij link do resetowania"
   - "Check your email" -> "Sprawdz swoja skrzynke"
   - "Invalid email or password" -> "Nieprawidlowy email lub haslo"

7. Populate `messages/en/auth.json` with the same keys and English strings (matching the current hardcoded text exactly).

8. Update `messages/pl/common.json` and `messages/en/common.json` if any additional common keys are needed (e.g., shared form labels not already in common.json).
