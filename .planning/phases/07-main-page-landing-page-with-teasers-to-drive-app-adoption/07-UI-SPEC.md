## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding (e.g. icon-to-text gap in feature cards) |
| sm | 8px | Compact element spacing (e.g. subheadline margin-top below headline) |
| md | 16px | Default element spacing (e.g. padding inside feature cards, CTA button padding-x) |
| lg | 24px | Section padding (e.g. hero vertical padding, feature cards internal padding) |
| xl | 32px | Layout gaps (e.g. gap between hero and features section) |
| 2xl | 48px | Major section breaks (e.g. hero top/bottom padding on desktop) |
| 3xl | 64px | Page-level spacing (e.g. full hero section vertical padding on large screens) |

Exceptions:
- Touch targets for CTA buttons: minimum 44px height (mobile accessibility standard)
- Feature card grid gap: 24px on desktop, 16px on mobile

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | 400 (regular) | 1.5 |
| Label | 14px | 400 (regular) | 1.5 |
| Heading | 24px | 700 (bold) | 1.2 |
| Display | 36px | 700 (bold) | 1.15 |

Notes:
- Display size (36px) is used exclusively for the hero headline "Replace the spreadsheet." — larger than any existing in-app heading to signal this is a marketing surface
- Heading (24px / `text-2xl`) matches existing in-app page heading pattern from `login/page.tsx`; also used for FAQ section headings "For Trainers" and "For Trainees"
- Body (16px / `text-base`) used for feature card descriptions and hero subheadline
- Label (14px / `text-sm`) used for footer links, FAQ section labels, nav links in app header
- Only two weights are used: 400 (regular) and 700 (bold)

Source: codebase scan of `login/page.tsx` and `NavHeader.tsx` for existing type scale; display size is Claude's discretion per CONTEXT.md.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#0f172a` (`bg-bg-page`) | Page background, hero section background, footer background |
| Secondary (30%) | `#1e293b` (`bg-bg-surface`) | Feature cards background, FAQ accordion/section backgrounds |
| Accent (10%) | `#10b981` (`accent`) | Primary CTA button fill, active nav link text, feature card icon color |
| Destructive | `#ef4444` (`error`) | Not used on this phase — no destructive actions on public pages |

Accent reserved for:
1. "Sign up as trainer" primary CTA button — background fill
2. Feature card icons (3 icons) — icon stroke/fill color
3. Active nav link in `NavHeader` and `TraineeNavHeader` for the new Help link (hover state only, consistent with existing pattern)
4. Hover state on footer links (`hover:text-accent-hover` → `#34d399`)

Text color: `#e2e8f0` (`text-text-primary`) for all body copy, headlines, and labels.
Border color: `#334155` (`border-border`) for card borders and horizontal rule separators.

"Join as trainee" ghost button: `border-border` border, `text-text-primary` text, no background fill — visually secondary to the accent-filled trainer CTA. Source: CONTEXT.md ("ghost/outline style vs filled accent button").

Source: CONTEXT.md (locked — "Same design system as the app"); `globals.css` confirms all token values.

## Page Layout Contract

### Landing Page (`/`)

Three sections in order — no other sections.

**Section 1: Hero**
- Full-width, vertically centered, `bg-bg-page`
- Vertical padding: 64px top / 64px bottom on desktop; 48px top / 48px bottom on mobile
- Max content width: 640px, centered via `mx-auto px-4`
- Layout (top to bottom):
  1. `ForgeLogo` horizontal variant, `h-10`, centered
  2. Headline: "Replace the spreadsheet." — display size (36px), weight 700, `text-text-primary`, `mt-8`, `text-center`
  3. Subheadline: "Trainers build plans. Trainees log workouts. In under a minute." — 16px, weight 400, `text-text-primary`, opacity 80%, `mt-4`, `text-center`
  4. CTA row: `mt-8`, flex row, `gap-4`, centered; on very small screens (`< 380px`) stacked column with `gap-3`
     - Button 1: "Sign up as trainer" → `/signup/trainer` — filled, `bg-accent text-white hover:bg-accent-hover`, min-height 44px, `px-6 py-3 rounded-sm font-bold text-base`
     - Button 2: "Join as trainee" → `/signup/trainee` — ghost, `border border-border text-text-primary hover:border-accent hover:text-accent`, same sizing as Button 1

Source: CONTEXT.md (locked headline, subheadline, CTA labels, button styles, routes).

**Section 2: Features**
- `bg-bg-page`, vertical padding 48px top / 48px bottom
- Optional section heading above cards: "Why Forge?" — 24px, weight 700, `text-text-primary`, `text-center mb-8` (Claude's discretion to include; recommended for scannability)
- 3-column grid on desktop (`md:grid-cols-3`), single column stacked on mobile
- Grid gap: 24px desktop / 16px mobile
- Max grid width: 960px, centered via `mx-auto px-4`

Each feature card:
- `bg-bg-surface border border-border rounded-sm p-6`
- Layout: icon (top) → title (below icon, `mt-4`) → description (below title, `mt-2`)
- Icon: inline SVG, 28px × 28px, `text-accent` (stroke color)
- Title: 16px, weight 700, `text-text-primary`
- Description: 14px, weight 400, `text-text-primary`, opacity 70%, max 15 words

Card content (locked from CONTEXT.md, descriptions are Claude's discretion):

| Card | Icon (inline SVG) | Title | Description |
|------|-------------------|-------|-------------|
| 1 | Clipboard/list icon | Build structured plans | Create multi-week programs with sets, reps, and weights. |
| 2 | Lightning bolt / timer icon | Log workouts in seconds | See last week's results inline and record today's sets fast. |
| 3 | Chart / trend-up icon | Track progress over time | Trainers monitor every session, trainees watch their lifts grow. |

Source: CONTEXT.md (locked card titles and intent); icons and descriptions are Claude's discretion.

**Section 3: Footer**
- `bg-bg-page border-t border-border`, padding `py-8 px-4`
- Content centered, `text-sm text-text-primary`, flex row with `gap-6` centered, stacked on very small screens
- Links:
  1. "Already have an account? Log in" → `/login` — `text-accent hover:text-accent-hover`
  2. "Help / FAQ" → `/help` — `text-text-primary hover:text-accent`

Source: CONTEXT.md (locked footer content and routes).
