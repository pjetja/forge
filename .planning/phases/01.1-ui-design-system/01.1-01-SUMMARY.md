## Decisions Made
- Used `@theme inline` (not plain `@theme`) because `--font-sans: var(--font-lato)` requires CSS custom property resolution — plain @theme breaks chained variable resolution
- Applied `lato.variable` to `<html>` element so `@theme inline` can resolve `var(--font-lato)` at root scope
- Explicitly specified `weight: ['400', '700']` because Lato is not a variable font
