# Phase 01.1: UI Design System - Context

**Gathered:** 2026-03-10
**Status:** Ready for planning

<domain>
## Phase Boundary

Redesign all existing pages with a dark navy color scheme and consistent visual language. Scope: auth pages (login, signup trainer, signup trainee, verify-email), app shells (trainer home, trainee home), and the join/[token] page. No new features — purely visual rework of pages that already exist.

</domain>

<decisions>
## Implementation Decisions

### Color Palette
- Page background: `#0f172a` (dark navy)
- Surface / card background: `#1e293b` (slightly lighter — creates visual lift)
- Border color: `#334155`
- Accent / brand color: Emerald green — `#10b981` at rest, `#34d399` on hover
- Error / destructive color: Standard red `#ef4444`, lighter variant `#f87171`
- No box-shadows on cards — border + background lift provides sufficient depth

### Component Style
- Corner radius: Sharp — `2–4px` (`rounded-sm` in Tailwind)
- Primary buttons: Solid filled — bg `#10b981`, white text, hover bg `#34d399`, no border
- Secondary actions: Ghost button — no background, muted text, faint border appears on hover
- Destructive actions (e.g. sign out if treated as destructive): Outlined red — `border #ef4444`, text `#ef4444`, transparent bg
- Form inputs: bg `#1e293b`, border `1px solid #334155`, focus border `#10b981`, static label above field, white text

### Typography
- Font: **Lato** (load via Google Fonts or next/font)
- Primary / body text: Off-white `#e2e8f0`
- Secondary / helper text: Same color as body (`#e2e8f0`), just smaller font size — no color muting
- Headings: Claude's discretion on weight and size scale using Lato — should establish clear hierarchy relative to body text

### Page Layout
- Auth pages (login, signup, verify-email): Full-page dark navy background (`#0f172a`), single centered card (`#1e293b`, border `#334155`)
- App shells (trainer home, trainee home, join page): Same dark navy bg, content in card/surface containers
- All pages get a **top navigation bar**: app logo (text + small emoji/icon, Claude picks) in accent green on left, sign-out on right. Dark navy bg, border-bottom `#334155`

### Claude's Discretion
- Exact Lato font weights to use (likely 400 regular + 700 bold)
- Heading font size scale (e.g. h1=24px/semibold, h2=20px/medium)
- Logo emoji/icon choice — pick something fitness-relevant that fits the minimal style
- Exact padding/spacing inside cards and nav
- Loading and focus ring styles

</decisions>

<specifics>
## Specific Ideas

- App currently has low contrast between font and background — this is the primary complaint driving this phase
- Sharp corners (2–4px) chosen deliberately — more technical/tool-like than rounded fitness apps
- No shadows on cards — keep it flat and clean with border + bg lift only
- Secondary help text stays body color but smaller (not muted gray) — ensures readability for all users

</specifics>
