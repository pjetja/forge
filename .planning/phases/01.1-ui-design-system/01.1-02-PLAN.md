Anti-patterns to avoid (from RESEARCH.md):
- NO rounded-lg, rounded-xl → use rounded-sm everywhere
- NO shadow-sm, shadow-lg → no shadows on cards
- NO bg-white → bg-bg-surface for cards
- NO text-gray-400, text-gray-500 → text-text-primary (all body text same color, size varies)
- NO bg-blue-600, text-blue-600 → bg-accent, text-accent
- NO focus:ring-2 focus:ring-blue-500 → focus:border-accent
- "or" divider span: bg-bg-surface (NOT bg-bg-page — it sits on the card)

  <name>Task 1: Restyle auth page wrappers (login, signup trainer, signup trainee, verify-email)</name>
  <files>
    src/app/(auth)/login/page.tsx,
    src/app/(auth)/signup/trainer/page.tsx,
    src/app/(auth)/signup/trainee/page.tsx,
    src/app/(auth)/verify-email/page.tsx
  </files>
  <action>
    All four auth page files follow the same structural pattern: a wrapper div containing a dark card with the page content. Apply these class replacements consistently across all four files:

    **Card container:** Replace `bg-white shadow-sm rounded-xl` (or similar) with `bg-bg-surface border border-border rounded-sm`. Remove any shadow-* classes.

    **Page heading (h1/h2):** Replace any `text-gray-900` with `text-text-primary`. Apply `text-2xl font-bold` for main headings, `text-xl font-bold` for subheadings. Lato 700 (loaded in Plan 01) provides the bold weight.

    **Subtext / description paragraphs:** Replace `text-gray-600`, `text-gray-500`, etc. with `text-text-primary text-sm`. Do NOT use opacity or muted colors — keep the same off-white, just smaller.

    **Links (e.g. "Already have an account? Sign in", "Resend email"):** Replace `text-blue-600 hover:text-blue-700` with `text-accent hover:text-accent-hover`. No underline needed by default.

    **verify-email/page.tsx specifically:** The "Resend confirmation email" link/button should become `text-accent hover:text-accent-hover`. Any checkmark icon or success indicator should stay — only color changes.

    Card padding: use `p-8` if not already set. Space between card sections: `space-y-6`.

    Example final card structure:
    ```tsx
    <div className="bg-bg-surface border border-border rounded-sm p-8 space-y-6">
      <div>
        <h1 className="text-2xl font-bold text-text-primary">Welcome back</h1>
        <p className="text-sm text-text-primary mt-1">Sign in to your account</p>
      </div>
      {/* form or children */}
      <p className="text-sm text-text-primary text-center">
        Don't have an account?{' '}
        <a href="/signup/trainer" className="text-accent hover:text-accent-hover">
          Sign up
        </a>
      </p>
    </div>
    ```
  </action>
  <verify>
    Run `npx next dev`. Open each page:
    - http://localhost:3000/login
    - http://localhost:3000/signup/trainer
    - http://localhost:3000/signup/trainee
    - http://localhost:3000/verify-email

    On each page: dark navy full-page bg visible, centered dark surface card with border, no white or gray areas, all text off-white, no blue links (should be emerald). No rounded corners larger than 4px. No shadows.
  </verify>
  <done>All 4 auth page wrappers show dark surface cards on dark navy background. No bg-white, no gray text, no blue links, no shadows, no large rounded corners remain in these files.</done>

<task type="auto">
  <name>Task 2: Restyle LoginForm and SignupForm shared components</name>
  <files>
    src/app/(auth)/login/_components/LoginForm.tsx,
    src/components/auth/SignupForm.tsx
  </files>
  <action>
    Both files contain similar structures: Google OAuth button, "or" divider, email/password inputs, primary submit button, and error state. Apply the dark design system to both files identically.

    **Form inputs (email, password, name fields):**
    Replace: `border-gray-300 rounded-lg focus:ring-2 focus:ring-blue-500 focus:border-transparent bg-white text-gray-900`
    With: `bg-bg-surface border border-border rounded-sm px-3 py-2 text-sm text-text-primary focus:outline-none focus:border-accent transition-colors w-full`

    **Input labels (static labels above fields):**
    Replace: `text-gray-700`, `text-gray-600`
    With: `text-text-primary text-sm font-medium`

    **Primary submit button:**
    Replace: `bg-blue-600 hover:bg-blue-700 rounded-lg text-white`
    With: `bg-accent hover:bg-accent-hover text-white rounded-sm px-4 py-2 text-sm font-medium transition-colors disabled:opacity-50 w-full`

    **Google OAuth button:**
    Replace: `border-gray-300 hover:bg-gray-50 bg-white text-gray-700 rounded-lg`
    With: `border border-border rounded-sm px-4 py-2 text-sm font-medium text-text-primary hover:border-accent transition-colors w-full flex items-center justify-center gap-2`
    Keep the existing Google SVG icon — its colors are fine on dark backgrounds.

    **"or" divider — CRITICAL:**
    The span inside the horizontal rule divider must match the CARD surface, not the page background:
    Replace: `bg-white px-2` (or `bg-gray-50`)
    With: `bg-bg-surface px-2`
    The outer border line: `border-t border-border` (replace `border-gray-200` or similar)

    **Error state messages:**
    Replace: `bg-red-50 text-red-600 border-red-200` (or similar light red)
    With: `text-sm text-error bg-error/10 border border-error/30 rounded-sm px-3 py-2`

    **Any remaining text in these components:**
    - Replace `text-gray-500`, `text-gray-600`, `text-gray-400` → `text-text-primary text-sm`
    - Replace `text-blue-600`, `hover:text-blue-700` → `text-accent hover:text-accent-hover`
    - Replace `rounded-lg` → `rounded-sm`
    - Remove any `shadow-*` classes

    **Pending/loading state:** If a spinner or "Loading..." text exists, keep it. Style spinner border as `border-white` on the emerald button background.
  </action>
